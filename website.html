LibraryDesk ‚Äî a compact, production-ready Online Library & Inventory System

A lightweight Python/Tkinter frontend with an optional Flask backend for managing books, magazines and media. Designed for clarity, portability and developer productivity ‚Äî ideal for small libraries, labs, and developer demos.

Get Started Download App View Gallery

Quick facts

Tech: Python ¬∑ Tkinter ¬∑ Flask (optional)
Mode: Desktop UI with optional local backend
Data: JSON file or mock backend for demos
Concept & Key Features
LibraryDesk is designed around three goals: simplicity for end-users, minimal dependencies for portability, and a clear API-driven separation of UI and backend for extensibility.

Catalog & Management

Add, update and remove items (books, magazines). Each item stores title, creator, publication year and category.

Optional Backend

Includes a small JSON-backed Flask server (mock) ‚Äî useful for local demos, testing and integration. Endpoints for list, search, create, delete.

Responsive Desktop UI

High-DPI Tkinter UI with clear typography, searchable catalog, sidebar add form and confirm-delete modal for safety.

Safe Deletion Flow

Delete requires explicit confirmation (typing 'b') to avoid accidental removals ‚Äî ideal for production demos or training sessions.

Seamless Sync

Frontend communicates using REST endpoints. The UI reloads the full list after create/delete to preserve consistent serial ordering.

Testable

Includes headless smoke test mode (`--no-gui`) and mock backend (`--mock`) so CI or dev machines can run quick checks without a GUI.

Architecture & Code Overview
The app uses a clean separation:
  • Frontend (Tkinter): User interactions, list rendering, add/delete workflows, local UI state.
  • Backend (Flask, optional): Small JSON-based REST API with endpoints for listing, filtering, searching, creation and deletion.
  • Storage: `media.json` for persistent data, or in-memory store for the mock demo server.

REST endpoints (example)

Endpoints implemented by the mock Flask backend:

  • /media ‚Äî GET (list all)
  • /media/category/<category> ‚Äî GET (filter)
  • /media/search/<name> ‚Äî GET (search)
  • /media/create ‚Äî POST (create item)
  • /media/delete/<id> ‚Äî DELETE (delete item)

Important design choices

  • Items are stored with stable IDs (UUID or numeric) and the UI requests a full reload to keep serial ordering contiguous.
  • Mock backend is optional: the UI works with a live backend URL or the provided mock server for demos.
  • Focus on portability: minimal Python deps (Flask + requests + Pillow for GIFs if used).

Install & Run (Production-ready instructions)
Follow these steps for a local production-like environment. The app is intentionally small-dependency to be runnable on any desktop with Python 3.8+.

1. Prerequisites

Requirements
  • Python 3.8 or newer
  • pip (Python package manager)
  • Optional: Visual Studio Code for editing and debugging

2. Install dependencies

        # create a venv (recommended)
        python -m venv .venv
        # activate it:
        # Windows: .venv\Scripts\activate
        # macOS/Linux: source .venv/bin/activate

        # install required packages (assuming requirements.txt lists Flask, requests, etc.)
        pip install -r requirements.txt

3. Start the application

        # Run the backend (production or mock):
        python backend.py

        # Run the GUI application:
        python app.py

Note: In the repository sample, `backend.py` is the Flask server. The GUI script `app.py` (or the single-file script you already have) connects to the backend via the configured URL (defaults to http://127.0.0.1:5000).

Headless / CI test

Run:

python app.py --no-gui --mock

This executes a smoke test that creates and deletes a test record against the mock backend.

Configuration

  • To change backend URL: set environment variable LIBRARY_BACKEND_URL or pass --backend-url.
  • The mock backend is started with the --mock flag.
Test Plan & QA
Short checklist to verify the app works as expected on a new machine.

Manual smoke tests

  • Start backend and confirm GET /media returns JSON.
  • Open the GUI and verify it loads the catalog.
  • Create an item via sidebar ‚Äî it appears in the list after refresh.
  • Select an item, open delete confirm, type b, delete ‚Äî ensure it disappears.
  • Search for an item by title and verify results.

Automated checks

  • Run the headless smoke test: python app.py --no-gui --mock.
  • CI job: run the smoke test and ensure exit code 0 before publishing new builds.

Common troubleshooting

  • Backend unreachable ‚Äî ensure the mock backend is running or set correct --backend-url.
  • Missing dependencies ‚Äî run pip install -r requirements.txt.
  • Permission issues ‚Äî ensure you run Python with sufficient rights to create `media.json` in the project folder.

LibraryDesk Backend Structure

The backend is a lightweight Flask application providing a REST API for data operations. It uses a local JSON file for persistent storage.

Project Root
.venv/ (Python virtual environment)
app.py (Tkinter GUI - The Frontend)
backend.py (Flask REST API - The Backend)
media.json (Persistent Data Store)
requirements.txt

Sample `media.json` Content

        [
            {
                "id": "e0a7b4f8-c2d1-4e6a-8b0d-3c2f1a4e5b6c",
                "title": "The Phoenix Project",
                "creator": "Gene Kim",
                "year": 2013,
                "category": "Book"
            },
            {
                "id": "f5d9c3b2-a1e0-4d7c-9f8b-2a3d4c5e6f7a",
                "title": "Clean Code",
                "creator": "Robert C. Martin",
                "year": 2008,
                "category": "Book"
            }
        ]

Key REST API Endpoints

        # LIST ALL (via GET /media)
        $ curl -X GET http://127.0.0.1:5000/media

        # CREATE NEW ITEM (via POST /media/create)
        $ curl -X POST -H "Content-Type: application/json" \
            -d '{"title":"New Book","creator":"Anon","year":2025,"category":"Book"}' \
            http://127.0.0.1:5000/media/create

        # DELETE ITEM (via DELETE /media/delete/)
        $ curl -X DELETE http://127.0.0.1:5000/media/delete/e0a7b4f8-c2d1-4e6a-8b0d-3c2f1a4e5b6c
